ProbNumDiffEq.EK0 — Type
EK0(; prior=:ibm, order=1, diffusionmodel=:dynamic, smooth=true)

Gaussian ODE filtering with zeroth order extended Kalman filter.

Currently, only the integrated Brownian motion prior :ibm is supported. For the diffusionmodel, chose one of [:dynamic, :dynamicMV, :fixed, :fixedMV, :fixedMAP].

See also: EK1

References:

  • M. Schober, S. Särkkä, and P. Hennig: A Probabilistic Model for the Numerical Solution of Initial Value Problems
  • F. Tronarp, H. Kersting, S. Särkkä, and P. Hennig: Probabilistic Solutions To Ordinary Differential Equations As Non-Linear Bayesian Filtering: A New Perspective

ProbNumDiffEq.IEKS — Method
IEKS(; prior=:ibm, order=1, diffusionmodel=:dynamic, linearize_at=nothing)

Gaussian ODE filtering with iterated extended Kalman smoothing. To use it, use solve_ieks(prob, IEKS(), args...) instead of solve(prob, IEKS(), args...), since it is implemented as an outer loop around the solver.

Currently, only the integrated Brownian motion prior :ibm is supported. For the diffusionmodel, chose one of [:dynamic, :dynamicMV, :fixed, :fixedMV, :fixedMAP]. Just like the EK1 it requires that the Jacobian of the rhs function is available.

See also: EK0, EK1, solve_ieks

References:

  • F. Tronarp, S. Särkkä, and P. Hennig: Bayesian ODE Solvers: The Maximum A Posteriori Estimate

ProbNumDiffEq.MAPFixedDiffusion — Type

Maximum a-posteriori Diffusion estimate when using an InverseGamma(1/2,1/2) prior

The mode of an InverseGamma(α,β) distribution is given by β/(α+1) To compute this in an on-line basis from the previous Diffusion, we reverse the computation to get the previous sum of residuals from Diffusion, and then modify that sum and compute the new Diffusion.

OrdinaryDiffEq.perform_step! — Function

Perform a step

Not necessarily successful! For that, see step!(integ).

Basically consists of the following steps

  • Coordinate change / Predonditioning
  • Prediction step
  • Measurement: Evaluate f and Jf; Build z, S, H
  • Calibration; Adjust prediction / measurement covs if the diffusion model "dynamic"
  • Update step
  • Error estimation
  • Undo the coordinate change / Predonditioning

\[m_{n+1}^P = A(h)*m_n P_{n+1}^P = A(h)*P_n*A(h) + Q(h)\]

ProbNumDiffEq.smooth — Method
smooth(x_curr::Gaussian, x_next_smoothed::Gaussian, Ah::AbstractMatrix, Qh::AbstractMatrix)

SMOOTH step of the (extended) Kalman smoother, or (extended) Rauch-Tung-Striebel smoother. It is implemented in Joseph Form:

\[m_{n+1}^P = A(h)*m_n P_{n+1}^P = A(h)*P_n*A(h) + Q(h) G = P_n * A(h)^T * (P_{n+1}^P)^{-1} m_n^S = m_n + G * (m_{n+1}^S - m_{n+1}^P) P_n^S = (I - G*A(h)) P_n (I - G*A(h))^T + G * Q(h) * G + G * P_{n+1}^S * G\]

ProbNumDiffEq.update! — Function
update!(x_out, x_pred, measurement, H, R=0)

UPDATE step in Kalman filtering for linear dynamics models, given a measurement Z=N(z, S). In-place implementation of update, saving the result in x_out.

\[K = P_{n+1}^P * H^T * S^{-1} m_{n+1} = m_{n+1}^P + K * (0 - z) P_{n+1} = P_{n+1}^P - K*S*K^T\]

Implemented in Joseph Form.

See also: predict
